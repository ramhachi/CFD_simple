# FSAE簡易CFDシステム - 概念設計書 (Conceptual Design)

本ドキュメントでは、FSAE向けの空力解析を、誰でも簡単に実行できるようにするためのシステム概念と全体設計について説明します。

## 1. システムの目的
本システムの目的は、**「STLファイルを所定のフォルダに置き、対話型のプログラムの質問に答えるだけで、メッシュ生成から解析、結果出力までを全自動で行うこと」**です。
OpenFOAMの複雑な設定ファイル群やC++ライクな構文を直接ユーザーに触れさせず、直感的な数値入力のみで解析を回せるようにします。

## 2. システムの基本コンセプト

### 2.1. Dockerを利用した環境非依存性
流体解析ソフトウェア「OpenFOAM」はインストール手順が複雑で環境依存のエラーが起きやすいのが課題です。
本システムでは **Docker** を活用し、公式の `openfoam-v2312-default` 等のイメージを使用することで、ホストマシン（Mac/Windows/Linux）を問わず、全く同じ環境で計算を実行します。

### 2.2. 動的なアセンブリ機能 (STLの配置自由化)
CADソフトから様々な空力部品や車両コンポーネントをエクスポートする際、原点位置が統一されていない（いわゆる設計規則が徹底されていない）という課題があります。
本システムは、**実行時に各部品の相対位置を組み立てる（アセンブリする）機能**を持ちます。

- **ユーザー入力による移動:** ユーザーは対話プログラム内で各STLに対する「X, Y, Z方向の移動量（オフセット）」をmm単位で指定できます。
- **グラウンドクリアランスの全体制御:** 最終的な車両モデル全体に対する「地面とのクリアランス（Z方向の追加オフセット）」も設定可能で、空気の流れに対する車高の影響を容易にシミュレーションできます。
- **フレーム・ドライバー等の構造物の考慮:** 空力部品（フロントウイング、ボディ等）だけでなく、力として集計しないが流れに影響を与える障害物（フレーム、ドライバー、サスペンション群）も別枠のSTL（`frame.stl`）として読み込み、適切な位置へ配置します。

### 2.3. タイヤの空力影響評価モード
FSAE車両におけるタイヤは、全抵抗の大きな割合を占めると同時に、後方への気流を乱す主要因です。
本システムは、指定された `tires.stl` を解析に含めるか含めないかを対話形式で選択できる「タイヤ空力影響考慮モード」を標準搭載します。これにより、タイヤの有無による空力パッケージへの影響差分を簡単に比較検討できます。

## 3. ユーザーの操作フローと体験 (UX)

ユーザーが実際に行う作業は以下の3ステップのみです。

1. **ファイルの準備:** 
   - 自分の設計したSTLファイル（`body.stl`, `front_wing.stl`など）
   - タイヤ（`tires.stl`）
   - フレームなど（`frame.stl`等）
   これらをシステムの実行フォルダ内に単純にドラッグ＆ドロップします。

2. **対話型スクリプトの実行:**
   - ターミナルで `python run_cfd.py` を実行します。
   - プログラムが聞いてくる以下の質問に答えます。
     - *「風速は何m/sにしますか？」*
     - *「計算にタイヤを含めますか？(y/n)」*
     - *「body.stl をタイヤ(基準)から何mm移動させますか？ (X, Y, Z)」*
     - *「フレーム(frame.stl) を何mm移動させますか？ (X, Y, Z)」*
     - *「車高（グラウンドクリアランス）は何mmにしますか？」*

3. **結果の確認:**
   - メッシングや複雑な流体計算はバックグラウンドのDocker内で全自動処理されます。
   - 完了後、ターミナル上に **抗力(Drag), 揚力(Lift), 空気抵抗係数(Cd), 揚力係数(Cl)** が分かりやすくテキスト表示されます。
   - `VTK` フォルダが自動生成されるため、ParaViewを用いてすぐに気流や圧力分布の可視化が行えます。

## 4. 全体アーキテクチャ概要

システムは大きく分けて「ホスト側の対話層」と「コンテナ側の解析自動化層」に分かれます。

- **Host層 (Python系):**
  ユーザーとの対話、入力値のバリデーション、入力値に基づいたOpenFOAM設定ファイル群（`controlDict`の風速、メッシュ設定など）への書き換え、STL移動用スクリプトの生成、そしてDockerコンテナの呼び出しを担います。
- **Container層 (OpenFOAM / Bash系):**
  ホスト層から渡された設定をもとに、仮想風洞の生成(`blockMesh`)、STLのアセンブリ(`surfaceTransformPoints`)、車体形状の切り抜き(`snappyHexMesh`)、流体計算本体(`simpleFoam`)、力係数の抽出機能、Paraview出力用の変換(`foamToVTK`)を一気通貫で実行します。
